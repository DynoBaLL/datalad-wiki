Glossary🔗

The Glossary provides short definitions of vocabulary:

datalad🔗

A data management tool that provides a unified interface for managing and sharing data across different platforms and storage systems. The Datalad Handbook provides comprehensive documentation and guides for using Datalad effectively.

datalad-next🔗

A datalad extension that provides additional features and improvements over the original datalad. Once installed, datalad-next replaces the original datalad command wwithout requiring any changes to existing workflows. datalad-next is recommended for all users, especially on Windows, where it provides better support and performance.

git🔗

A distributed version control system that allows multiple people to collaborate on a project by tracking changes to files and coordinating work on those files. Git is widely used in software development and other collaborative projects to manage code and track changes over time.

git-annex🔗

A tool that allows you to manage large files with Git without storing the file contents in the Git repository. Instead, git-annex stores metadata about the files and their locations, allowing you to track and manage large files efficiently. git-annex can connect to various storage backends, including local disks, network storage, and cloud services, making it a versatile tool for managing large datasets in a distributed environment.

github🔗

A web-based Git repository hosting service that hosts code repositories and provides collaborative development features. GitHub is widely used in the open-source community and offers a platform for sharing code and collaborating on projects. Github is hosted at .

gitlab🔗

A web-based Git repository hosting service that hosts code repositories and provides collaborative development features. The University of Geneva has its own GitLab instance, which is used for hosting code repositories and collaborative development: GitLab UNIGE.

HPC🔗

High-Performance Computing: a type of computing that uses powerful processors and parallel processing to perform complex computations quickly. HPC systems are often used in scientific research and other fields that require large-scale data processing and analysis. The University of Geneva has its own HPC system. More details can be found on the University of Geneva’s HPC page.

NAS🔗

Network Attached Storage: a storage device that can be accessed over a network, allowing multiple users to store and retrieve data. The University of Geneva has its own NAS storage service. More details can be found on the University of Geneva’s NAS storage page.

S3🔗

Amazon Simple Storage Service: a scalable object storage developed by Amazon Web Services (AWS) that provides a simple web services interface for storing and retrieving any amount of data from anywhere on the web. The University of Geneva has its own S3-compatible storage service, which can be used for storing and sharing data. More details can be found on the University of Geneva’s S3 storage page.

ssh🔗

Secure Shell: a cryptographic network protocol for operating network services securely over an unsecured network. SSH is commonly used for remote login and command execution on servers and other remote systems.

VM🔗

Virtual Machine: a software emulation of a physical computer that runs an operating system and applications just like a physical computer.

WSL🔗

Windows Subsystem for Linux: a compatibility layer for running Linux binary executables natively on Windows.
